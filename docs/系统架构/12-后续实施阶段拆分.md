# 12-后续实施阶段拆分

## 1. 阶段拆分原则

实施应按可运行的纵向能力推进，每个阶段都能形成可验证的用户价值。不要先完成所有底层再做页面，也不要先做全部页面再补业务规则。

## 2. 阶段一：基础用户与账本闭环

目标：用户可以登录并完成基础收入支出记账。

范围方向：

1. 用户认证与资料。
2. 默认分类与自定义分类。
3. 账户管理。
4. 手动收入/支出账单。
5. 账单列表、筛选和删除。
6. 基础仪表盘摘要。

## 3. 阶段二：预算、储蓄目标与提醒

目标：用户可以管理预算、储蓄目标和周期账单提醒。

范围方向：

1. 周、月、年和自定义周期预算。
2. 预算进度。
3. 储蓄目标。
4. 周期账单提醒规则。
5. 确认周期账单入账。
6. 站内通知。

## 4. 阶段三：统计可视化

目标：用户可以从图表中理解自己的消费结构。

范围方向：

1. 收支趋势。
2. 分类占比。
3. 分类排行。
4. 消费热力图。
5. 多币种统计展示。
6. 仪表盘体验优化。

## 5. 阶段四：AI 记账与 AI 分析

目标：AI 能辅助用户完成记账和财务理解。

范围方向：

1. 自然语言记账解析。
2. 解析结果确认入账。
3. 月度财务总结。
4. 消费行为分析。
5. 预算建议。
6. 趋势预测。
7. 问答式财务助手。
8. AI 风险提示。

## 6. 阶段五：导出与异步任务完善

目标：用户可以导出月报，并稳定查看异步任务状态。

范围方向：

1. Excel 月报导出。
2. PDF 月报导出。
3. 导出任务状态。
4. 导出历史。
5. AI 任务状态统一化。
6. 任务失败重试与通知。

## 7. 阶段六：实时通信增强，视 v1 反馈决定

目标：在确有体验收益时启用 WebSocket。

适合启用的触发条件：

1. AI 任务和导出任务等待时间较长，轮询体验不足。
2. 用户对实时提醒有明确需求。
3. 多端同步成为主要使用场景。
4. 后续进入家庭共享账本阶段。

启用方向：

1. `/ws/` 通道。
2. 实时通知。
3. 后台任务进度推送。
4. 多端刷新信号。

## 8. 后续扩展阶段

可在 v1 稳定后评估：

1. 微信/支付宝账单导入。
2. OCR 小票识别。
3. 家庭共享账本。
4. 信用卡与还款管理。
5. 资产负债管理。
6. 多币种汇率换算和历史汇率。
7. 更完善的隐私、注销和数据删除能力。
