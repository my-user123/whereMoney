# 10-Nginx反向代理设计

## 1. 入口目标

Nginx 作为系统统一入口，负责转发前端页面、后端 API 和后续可能启用的 WebSocket 连接。

## 2. 路由方向

```text
/       -> frontend
/api/   -> backend
/ws/    -> backend WebSocket endpoint，v1 可预留
```

## 3. 前端转发

前端应用由 Nginx 暴露给用户。移动端优先的页面路由由前端应用处理，Nginx 负责静态资源、页面入口和基础缓存策略。

## 4. API 转发

所有后端 REST API 通过 `/api/` 转发到后端服务。Nginx 不承担业务鉴权，业务鉴权由后端完成。

## 5. WebSocket 转发预留

v1 暂不强制启用 WebSocket，但 Nginx 设计中保留 `/ws/` 路由。后续启用后，该路径用于实时通知、AI 任务进度、导出任务进度和多端状态同步。

## 6. 安全与限制

1. 限制请求体大小，避免异常上传或导出请求滥用。
2. 保护后端服务不直接暴露到公网。
3. 对 API 和登录相关路径预留限流能力。
4. 不在访问日志中记录敏感令牌和完整敏感查询内容。

## 7. 静态与导出文件

导出文件不建议作为无鉴权静态文件直接暴露。应由后端校验权限后提供下载，或生成短期授权访问地址。
