# 02-前后端模块划分

## 1. 划分原则

前端负责用户体验、交互状态、图表展示、表单校验提示和页面级状态组织。后端负责认证鉴权、业务规则、数据持久化、统计聚合、AI 调用编排、导出任务和通知生成。

前后端通过 REST API 交互，后续如启用 WebSocket，仅用于通知和后台任务状态推送，不承载核心写入业务。

## 2. 前端模块

### 2.1 认证与用户设置

负责注册、登录、用户资料、默认货币、个人偏好等页面和交互。

### 2.2 首页仪表盘

负责展示本期收入、支出、结余、预算进度、储蓄进度、分类概览、趋势摘要和 AI 洞察入口。

### 2.3 记账模块

负责手动收入/支出记账、账单编辑、账单删除、账单筛选、账单列表和账单详情。

### 2.4 AI 记账与财务助手

负责自然语言输入、AI 解析结果确认、AI 问答、月度总结触发、预算建议触发和趋势预测展示。

### 2.5 分类与账户管理

负责账户列表、账户编辑、分类列表、自定义分类、分类启停和基础配置。

### 2.6 预算与储蓄目标

负责预算创建、预算进度展示、预算提醒查看、储蓄目标创建和储蓄目标进度展示。

### 2.7 周期账单提醒

负责周期提醒规则管理、到期提醒展示、确认入账、跳过本次和修改后入账。

### 2.8 统计与图表

负责折线图、饼图、热力图、分类排行、多币种统计展示和筛选条件交互。

### 2.9 通知中心

负责展示站内通知，包括预算提醒、周期账单提醒、月度总结提醒、导出完成和 AI 任务完成。

### 2.10 月报导出

负责导出入口、导出任务状态展示、文件下载和导出历史展示。

## 3. 后端模块

### 3.1 认证与用户服务

负责用户身份、登录状态、用户资料和默认设置。

### 3.2 账单服务

负责收入/支出账单的创建、编辑、删除、查询和账单来源管理。

### 3.3 账户服务

负责账户创建、账户编辑、账户归档和账户余额维护。

### 3.4 分类服务

负责默认分类、用户自定义分类、分类状态和分类展示配置。

### 3.5 预算服务

负责预算规则、预算进度计算、预算状态和预算提醒触发。

### 3.6 储蓄目标服务

负责储蓄目标管理、目标进度计算和目标状态维护。

### 3.7 周期提醒服务

负责周期账单提醒规则、到期检测、提醒生成、确认入账和跳过处理。

### 3.8 统计服务

负责仪表盘数据、趋势统计、分类统计、热力图数据、排行数据和统计缓存管理。

### 3.9 AI 服务编排

负责自然语言记账解析、报告生成、预算建议、趋势预测、财务问答和风险提示拼装。

### 3.10 通知服务

负责站内通知生成、读取、未读计数和后续 WebSocket 推送扩展。

### 3.11 导出服务

负责 Excel/PDF 月报生成、导出任务状态、文件管理和下载授权。

## 4. 前后端职责边界

1. 前端可以做轻量输入校验，但后端必须进行最终校验。
2. 前端不直接拼接财务统计结论，关键统计口径由后端统一提供。
3. 前端不保存用户敏感财务数据作为长期状态。
4. 后端不返回数据库实体对象，应返回面向页面的响应模型。
5. AI 解析结果必须由前端展示给用户确认，再由后端生成正式账单。
6. 删除账单后的统计失效、报告失效由后端负责。
