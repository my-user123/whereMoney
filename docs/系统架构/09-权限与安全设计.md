# 09-权限与安全设计

## 1. 安全目标

系统处理个人财务数据，安全目标包括身份可信、数据隔离、敏感信息保护、操作可追踪和 AI 权限可控。

## 2. 认证设计

第一阶段只有普通用户角色。用户登录后通过令牌访问受保护接口。后端从认证上下文识别用户身份，并用于所有业务查询和写入。

## 3. 授权边界

1. 用户只能访问自己的账户、分类、账单、预算、储蓄目标、提醒、AI 报告和导出记录。
2. 用户不能通过参数猜测访问其他用户资源。
3. 所有资源操作必须校验归属。
4. 第一阶段不提供后台管理员，不提供家庭成员权限分级。

## 4. AI 权限边界

1. AI 只能访问当前用户授权范围内的账本数据。
2. AI 自然语言解析结果必须由用户确认后才能入账。
3. AI 不能自动修改账单、预算、储蓄目标和账户。
4. AI 输出必须带风险提示，不能承诺收益或替代专业建议。
5. 用户手动修改 AI 解析结果后，系统第一阶段不进行长期偏好学习。

## 5. 数据安全

1. 密码等认证敏感信息必须加密或哈希存储。
2. 财务金额、账单备注、AI 问答内容和导出文件都视为敏感数据。
3. 导出文件需要访问控制，避免公开可猜测链接。
4. 日志中不应打印敏感财务明细、令牌或密钥。

## 6. 操作风险控制

1. 删除账单属于影响统计和报告的操作，需要明确确认。
2. 删除账单后，相关 AI 报告应删除或标记失效。
3. 周期账单提醒不能自动入账，必须由用户确认。
4. 多币种统计需要明确展示口径，避免误导用户。

## 7. WebSocket 安全预留

如后续启用 WebSocket：

1. 连接建立时必须校验用户身份。
2. 推送事件只能发送给资源所属用户。
3. WebSocket 不作为权限判断依据。
4. 用户离线时消息必须落入站内通知。
