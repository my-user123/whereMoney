# 04-前端目录结构

## 1. 目录结构原则

前端采用移动端优先的应用结构。页面按业务路由组织，复用组件、请求封装、类型定义、状态管理和工具函数独立放置。业务页面不直接散落请求逻辑，尽量通过 feature 模块聚合页面、组件、hooks 和类型。

## 3. 页面模块

### 3.1 首页仪表盘

展示收支概览、预算进度、储蓄目标、趋势摘要、分类占比和 AI 洞察入口。

### 3.2 账单页面

提供账单列表、筛选、手动记账、编辑、删除和详情查看。

### 3.3 AI 助手页面

提供自然语言记账、解析确认、AI 问答、月度总结、预算建议和趋势预测入口。

### 3.4 预算与储蓄页面

提供预算管理、预算进度、预算提醒和储蓄目标管理。

### 3.5 周期账单提醒页面

提供提醒规则管理、到期提醒、确认入账、跳过和修改后入账。

### 3.6 统计页面

提供趋势、分类占比、热力图、排行、多币种展示和筛选。

### 3.7 通知与导出页面

提供站内通知、导出任务状态、导出历史和文件下载。

## 4. 状态管理边界

1. 服务端数据由查询缓存管理。
2. 登录态、轻量全局 UI 状态和用户偏好可放入客户端状态。
3. 财务明细数据不应长期保存在客户端全局状态。
4. AI 对话输入和临时确认状态属于页面或 feature 内部状态。

## 5. WebSocket 前端预留

v1 如不启用 WebSocket，前端通过请求刷新和轮询获取导出任务、AI 任务和通知状态。目录中可预留实时连接封装位置，后续启用 WebSocket 后由通知中心、任务状态组件和仪表盘刷新逻辑复用。
