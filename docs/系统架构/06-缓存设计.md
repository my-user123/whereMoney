# 06-缓存设计

## 1. 缓存使用原则

缓存只用于提升读取性能、保存临时状态和支撑限流，不作为财务事实来源。核心账单、账户、预算、报告和通知数据以数据库为准。

## 2. 适合缓存的内容

1. 首页仪表盘摘要。
2. 常用统计结果。
3. 分类列表和账户列表。
4. 未读通知数量。
5. 导出任务和 AI 任务的临时进度。
6. 登录和接口限流计数。
7. 短期验证码或临时令牌，若后续需要。

## 3. 不适合缓存为事实的内容

1. 正式账单明细。
2. 账户余额事实。
3. 预算配置事实。
4. AI 报告原文事实。
5. 导出文件授权事实。

这些数据可以缓存副本，但必须能从数据库重新构建。

## 4. 缓存失效原则

以下操作发生后，应使相关缓存失效：

1. 新增、编辑、删除账单。
2. 修改账户。
3. 修改分类。
4. 修改预算。
5. 修改储蓄目标。
6. 确认周期账单入账。
7. 删除或重生成 AI 报告。

## 5. 任务状态缓存

AI 分析和月报导出可能需要展示进行中状态。v1 可将任务状态同时保存到数据库和缓存中：数据库用于最终状态追踪，缓存用于高频状态查询。

## 6. WebSocket 预留关系

如果后续启用 WebSocket，缓存可用于保存连接用户的在线状态、短期推送状态和任务进度快照。但在线状态不应成为业务权限判断依据。
